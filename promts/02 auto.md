Добавь поддержку необязательных параметров для команды /tr

subscribe — При True программа автоматически отправляет сообщение /tr на последующие новые сообщения в ответ на любые медиа в конкретном этом чате (видео, музыка, аудиосообщения, видеосообщения). При False программа перестаёт автоматически отправлять сообщения /tr в этом чате. При None режим подписки не меняется. По-умолчанию None.

Программа должна хранить список чатов и режим подписок в файле, который выживет после docker compose down и docker compose up. Формат сохранения: json.

subscribe_record_audio — True/False. Подписка на автоматическую отправку на аудиосообщения в чате.

subscribe_record_video — True/False. Подписка на автоматическую отправку на видеосообщения в чате.

subscribe_audio — True/False. Подписка на автоматическую отправку на музыку в чате.

subscribe_video — True/False. Подписка на автоматическую отправку на видео в чате.

Если несколько раз подписаться на один и тот же чат, то это не даст эффекта. Например если отправить сначала subscribe=True, а затем subscribe_record_audio=True, то сообщения не будут обработаны подписками дважды в раках чата.

Комбинация четырёх параметров subscribe_record_audio=True subscribe_record_video=True subscribe_audio=True subscribe_video=True это то же самое, что и subscribe=True.

destruct_message — True/False. При True не выполняет операции скачивания, конвертации и транскрипции. Удаляет отправленное сообщение. Удобно для подписки. По-умолчанию False.

help — True/False. При True выводит всю справку по аргументам и доступные команды. Не выполняет операции скачивания, конвертации и транскрипции. По-умолчанию False.
